**结果分析**

### 一、评估设置说明

本次实验在 **peoples_daily** 数据集（PER / ORG / LOC 三类实体，BIO 标注）上对比五种方案。需要注意两类指标口径：

| 方案 | 评估集 | 指标 |
|------|--------|------|
| BERT + Linear / CRF | validation 全量（2318 句，3451 个实体） | seqeval entity-level F1 |
| LLM API / Qwen2 SFT | validation 分层采样 100 条（seed=42） | span F1（实体文本 + 类型 + 字符位置完全匹配） |

因此，BERT 与 LLM/SFT 的 F1 **不宜直接当作同口径数值比较**，但趋势和相对优劣仍有参考价值。下文分方案讨论。

---

### 二、BERT 序列标注：Linear vs CRF

**整体表现**

两种 BERT 方案在 validation 全量上均达到 **F1 > 0.95**，说明 BERT 预训练表征与 BIO 序列标注范式对该新闻 NER 任务非常匹配。

| 方案 | F1 | 非法 BIO 序列 |
|------|-----|---------------|
| BERT + Linear | 0.9516 | 47 条（2.0%） |
| BERT + CRF | 0.9542 | 32 条（1.4%） |

CRF 相比 Linear 头 F1 提升 +0.0027，绝对值不大，但在已接近上限（0.95+）的情况下仍有稳定增益。更明显的差异在 标签合法性：CRF 通过转移矩阵约束 BIO 转移，非法序列从 47 条降至 32 条（减少约 32%），说明 CRF 能在解码阶段抑制「I-ORG 接在 B-PER 之后」这类结构错误。此外，当前 CRF 仍剩 32 条非法序列，与训练日志中「转移矩阵尚未完全收敛」的提示一致；若进一步增加训练轮次，转移矩阵学习更充分，非法序列有望继续下降。

**逐类型分析**

| 类型 | Linear F1 | CRF F1 | 说明 |
|------|-----------|--------|------|
| PER（人名） | 0.9833 | 0.9833 | 两者持平，识别最稳定 |
| LOC（地名） | 0.9570 | 0.9599 | CRF 略优 |
| ORG（机构） | 0.9136 | 0.9182 | 两类方案中最难，CRF 提升相对明显 |

ORG 的 F1 明显低于 PER 和 LOC，可能因为机构名边界模糊（如「XX公司XX部」）、嵌套结构多、与 LOC 共现时易混淆。PER 形态相对固定，故两者均接近 0.98。

**小结**：在专用 NER 任务上，BERT + CRF 是当前最优方案——精度高、解码合法、三类实体均衡。Linear 头已足够强，CRF 主要补「标签结构约束」这一短板。

---

### 三、LLM API（DeepSeek）：zero-shot vs few-shot

| 方案 | Precision | Recall | F1 |
|------|-----------|--------|-----|
| zero-shot | 0.7704 | 0.6520 | 0.7063 |
| few-shot（3 例） | 0.7889 | 0.6677 | 0.7233 |

**主要观察**

1. **无需微调即可用**：DeepSeek API 在 100 条采样上 F1 约 0.71，说明大模型预训练已具备一定中文 NER 能力，配合 JSON 结构化 prompt 可直接部署。
2. **few-shot 略优于 zero-shot**（F1 +0.017）：3 个标注样例帮助模型对齐输出格式（PER/ORG/LOC 英文标签）和边界习惯，precision 提升更明显（0.77 → 0.79）。
3. **Recall 是瓶颈**（约 0.65–0.67）：模型倾向于保守预测或边界偏移（如「学昭女士」vs gold「学昭」），导致漏检和部分 span 不匹配。
4. **与 BERT 的差距**：参考 F1 相差约 0.23，但 BERT 为全量 seqeval、LLM 为 100 条 span F1，实际差距可能略小；即便如此，专用序列标注模型在 NER 精度上仍明显领先。

**适用场景**：快速原型、无标注数据、多任务统一接口；不适合对 F1 和延迟有严格要求的生产 NER 流水线。

---

### 四、Qwen2-0.5B SFT（LoRA）

| 指标 | 数值 |
|------|------|
| Precision | 0.1534 |
| Recall | **0.0909** |
| F1 | 0.1142 |
| JSON 解析失败 | 0 条 |
| 推理耗时 | ~19.3 s/条（CPU） |

SFT 结果远低于 LLM API（F1 0.11 vs 0.72），也远低于 BERT。JSON 能解析，说明模型学会了输出 JSON 外壳，但 **实体识别质量很差**。

**原因分析**

1. **召回崩溃**：Recall 仅 0.09，100 条中有大量样本 pred 为空。常见情况是模型输出中文类型标签（如「人名」「组织机构」）而非 PER/ORG/LOC，评估脚本过滤后等价于未预测任何实体。
2. **模型容量小**：Qwen2-0.5B 参数量远小于 DeepSeek API 所用模型，细粒度 span 边界学习更难。
3. **训练不充分**：`train_sft.json` 显示仅完成 **1 个 epoch**（全量 20864 条，CPU 约 7 小时），默认 3 epoch 未跑完，LoRA 尚未充分收敛。
4. **边界与类型错误**：部分样本出现过度切分（如将「马克思主义」「世界」标为 PER）或类型混淆（LOC 标成 ORG），进一步拉低 precision。

**与 API few-shot 对比**：F1 低 0.61，说明 **小模型 + 不充分 SFT ≠ 大模型 + prompt**；在当前训练条件下，本地 SFT 不具备替代 API 或 BERT 的条件。

**改进方向**（若继续实验）：增加 epoch、用 `--num_train 2000` 快速迭代、提高 `max_length` 减少截断、评估侧增加中英文 type 映射。

### 五、综合对比与结论

```
方案                    F1（参考）    优势                          劣势
─────────────────────────────────────────────────────────────────────────
BERT + CRF              0.9542       精度最高、BIO 合法、可离线      需标注数据训练
BERT + Linear           0.9516       实现简单、与 CRF 接近           非法 BIO 略多
LLM API few-shot        0.7233       零训练、prompt 可调             成本高、Recall 偏低
LLM API zero-shot       0.7063       最简单                          格式/边界更不稳定
Qwen2-0.5B SFT          0.1142       可本地部署                      当前训练不足、效果差
```

**核心结论**

1. **任务专用模型最优**：BERT + CRF 在人民日报 NER 上 F1 达 0.954，CRF 在 F1 和 BIO 合法性上均优于 Linear，验证了序列标注层对结构化预测的价值。
2. **大模型 API 是可行备选**：DeepSeek few-shot F1 ~0.72，无需微调即可使用，适合探索或无标注场景；但 precision/recall 均衡性不如 BERT，且依赖网络与 API 成本。
3. **小模型 SFT 需要充分训练**：0.5B + 1 epoch LoRA 远未收敛，不能代表 SFT 方法的上限；当前结果说明 **模型规模、训练轮次、输出格式约束** 对生成式 NER 至关重要。
4. **方法选型建议**：
   - 追求精度、有标注数据 → **BERT + CRF**
   - 快速验证、无训练资源 → **LLM API + few-shot prompt**
   - 本地私有化 LLM → 需更大模型或更充分 SFT，当前 Qwen2-0.5B 配置不满足生产要求

- LLM / SFT 仅在 100 条上评估，与 BERT 全量结果口径不同，跨方案 F1 对比为近似参考。
- SFT 在 CPU 上训练与推理，epoch 与超参受时间限制，未探索更优配置。
